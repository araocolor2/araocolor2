# 제주컬러 UI/UX 인터뷰

> 모바일 기준, 최신 트렌드 디자이너 관점의 질문과 답변 기록

---

## Q1. 홈 화면 — 스크롤 없이 보이는 첫 화면 배치

스크롤 없이 보이는 첫 화면에 무엇을 배치할까요?

> ① 이미지 + 슬로건만 — 감성적 브랜딩 우선, 상품은 스크롤 후
> ② 이미지 + 슬로건 + 대표 상품 1개 바로 노출 — 이미지 아래 즉시 구매 유도
> ③ 이미지를 줄이고 상품 카드 2개를 바로 노출 — 쇼핑몰 느낌

**A: ①** 이미지 + 슬로건만 (감성적 브랜딩 우선)

---

## Q2. 하단 탭바 — 아이콘과 텍스트

탭바 스타일을 어떻게 할까요?

> ① 아이콘 + 텍스트
> ② 아이콘만
> ③ 현재 선택된 탭만 텍스트 표시

**A: 아이콘만 + 마우스오버 시 텍스트 표시 (웹)**

---

## Q3. 상품 카드 — 목록에서 보이는 정보

각 카드에 어떤 정보를 보여줄까요?

> ① 이미지 + 상품명 + 가격
> ② 이미지 + 상품명 + 가격 + 별점/후기 수
> ③ 이미지 + 상품명 + 가격 + 짧은 한 줄 설명

**A: ③** 이미지 + 상품명 + 가격 + 짧은 한 줄 설명

---

## Q4. 상품 상세 페이지 — 구매 버튼 위치

구매하기 버튼을 어디에 배치할까요?

> ① 상단 고정 바 — 스크롤해도 항상 화면 아래 붙어있음
> ② 이미지 바로 아래
> ③ 상품 설명 다 읽은 후 하단

**A: ①** 하단 고정 바 (전문가 추천 — 애플, 무신사, 크림 표준 방식)

---

## Q5. 상품 상세 페이지 — 미리보기 방식

구매 전 상품을 어떻게 미리 보여줄까요?

> ① 샘플 이미지 여러 장 (갤러리)
> ② 샘플 이미지 + 일부 흐리게 처리
> ③ 짧은 미리보기 영상

**A: 비포/애프터 슬라이더** — 샘플 이미지와 비교 이미지 두 장을 슬라이드로 움직이며 변화를 보여줌

---

## Q6. 상품 상세 페이지 — 스크롤 콘텐츠 순서

위에서 아래로 콘텐츠 순서는?

> ① 미리보기 → 상품 설명 → 가격 → 후기 → 구매버튼(고정)
> ② 미리보기 → 가격 → 상품 설명 → 후기 → 구매버튼(고정)
> ③ 미리보기 → 후기 → 상품 설명 → 가격 → 구매버튼(고정)

**A: ②** 미리보기 → 가격 → 상품 설명 → 후기 → 구매버튼(고정) (전문가 추천 — 가격을 뒤에 두면 숨기는 느낌)

---

## Q7. 로그인 모달 — 버튼 배치

모달 안 구성은?

> ① 버튼만 — "구글로 로그인" 버튼 하나만
> ② 한 줄 안내 + 버튼
> ③ 혜택 강조 + 버튼

**A: ①** 버튼만

---

## Q8. 아이디 등록 화면 — 안내 문구

첫 로그인 직후 아이디 등록 화면 상단 안내는?

> ① 안내 없이 입력창만
> ② 한 줄 안내 — "커뮤니티에서 사용할 닉네임을 정해주세요"
> ③ 안내 + 규칙 표시

**A: ②** 한 줄 안내만

---

## Q9. 구매 흐름 — 단계 표시

SMS 인증 → 결제 진행 시 단계 표시가 필요할까요?

> ① 필요 없다 — Stripe 결제창으로 바로 이동
> ② 간단한 단계 표시 — "인증 → 결제" 2단계
> ③ 진행 바

**A: ①** 단계 표시 없이 바로 이동

---

## Q10. 구매 완료 화면

결제 후 바로 보이는 화면은?

> ① 완료 화면 없이 마이페이지로 바로 이동
> ② 간단한 완료 화면 후 마이페이지로 이동
> ③ 완료 화면에서 다운로드 버튼 바로 제공

**A: ②** "결제 완료! 이메일을 확인하세요" 완료 화면 → 마이페이지 이동 버튼

---

## Q11. 마이페이지 — 구매 내역 카드 디자인

구매 내역 표시 방식은?

> ① 바로 보임 — 상품명 + 구매일 + 다운로드 버튼 한 줄 표시
> ② 눌러서 펼침 — 상품명만 보이고 누르면 상세 + 다운로드 버튼 펼쳐짐
> ③ 카드 형태

**A: ②** 눌러서 펼치는 방식

---

## Q12. 게시판 목록 — 글 카드에 보이는 정보

각 글 카드에 표시할 정보는?

**A: 직접 답변**
- 제목 + 작성자 + 날짜 + 댓글 수 + 좋아요 수
- 이미지가 있을 경우 첫 이미지 썸네일
- 파일이 첨부된 경우 파일 아이콘
- 텍스트만 있는 경우 텍스트 아이콘

---

## Q13. 글 상세 페이지 — 댓글 입력창 위치

댓글 입력창 위치는?

벨 아이콘 배치는?

> ② 하단 탭바 마이페이지 탭에 뱃지

빈 화면 처리는?

> ③ 일러스트 + 텍스트 + 행동 유도 버튼

**A: ③** 일러스트 + 텍스트 + 행동 유도 버튼 ("첫 글을 작성해보세요")

## Q16. 오류 화면 — 다운로드 링크 만료

7일 만료 링크 접근 시 화면은?

> ① 단순 오류 메시지
> ② 안내 + 마이페이지 이동 버튼
> ③ 안내 + 로그인 유도

**A: ②** "링크가 만료되었습니다. 마이페이지에서 다시 다운로드하세요" + 버튼

## Q17. 상품 후기 — 목록에서 보이는 방식

후기 표시 방식은?

> ① 별점 + 텍스트 목록
> ② 전체 평균 별점 크게 + 후기 목록
> ③ 후기 없을 때는 숨김

**A: ②** 전체 평균 별점 크게 + 후기 목록 (전문가 추천 — 아마존, 쿠팡 표준 방식)

## Q18. 고객 문의 — 제출 후 화면

문의 폼 제출 후 화면은?

> ① 완료 메시지만
> ② 완료 화면으로 전환 + 응답 예상 시간 안내
> ③ 문의 내역 페이지로 이동

**A: ②** 완료 화면 전환 + "영업일 기준 1~2일 내 답변" 안내 (전문가 추천 — 표준 방식)

> ① 스크롤 위치 유지
> ② 맨 위로 이동
> ③ 상관없다

**A: ①** 스크롤 위치 유지

## Q20. 전체 색상 — 강조 색상

구매하기, 다운로드 버튼 및 알림 뱃지 강조 색상은?

> ① 진한 청록 — 제주 바다 느낌, 차분하고 신뢰감
> ② 선명한 파랑 — 깔끔하고 모던
> ③ 따뜻한 오렌지/산호 — 파랑 배경과 대비

**A: ①** 진한 청록

---

## Q21. 결제 완료 후 다음 행동 유도

결제 완료 화면에서 다음 행동 버튼은 어떻게 둘까요?

> ① 마이페이지 이동만 제공
> ② 마이페이지 이동 + 제주컬러 전용 게시판 보기 버튼
> ③ 홈으로 이동만 제공

**A: ②** 마이페이지 이동 + 전용 게시판 보기 버튼 (결제와 커뮤니티 흐름 연결)

## Q22. 환불 후 전용 게시판 접근 시 화면

환불 완료 회원이 전용 게시판에 접근하면 어떻게 보여줄까요?

> ① 접근 불가 텍스트만 표시
> ② 안내 문구 + 상품 페이지 이동 버튼
> ③ 로그인 화면으로 강제 이동

**A: ②** "이용 권한이 변경되었습니다" 안내 + 상품 페이지 이동 버튼

## Q23. 커뮤니티 테스트 시 우선 확인할 알림 동작

댓글/좋아요 알림 테스트 기준은?

> ① 알림 개수만 확인
> ② 알림 생성 + 뱃지 숫자 + 클릭 후 읽음 처리까지 확인
> ③ 알림 목록 화면만 확인

**A: ②** 생성/뱃지/읽음 처리까지 한 흐름으로 확인

---

## Q24. 결제+커뮤니티 통합 테스트 순서

통합 테스트는 어떤 순서가 가장 안전할까요?

> ① 결제만 먼저 끝낸 뒤 커뮤니티 별도 테스트
> ② 커뮤니티만 먼저 테스트 후 결제 연결
> ③ 결제 단독 테스트 → 커뮤니티 단독 테스트 → 권한 연동 테스트

**A: ③** 결제 단독 → 커뮤니티 단독 → 권한 연동 (구매 후 접근/환불 후 차단)
