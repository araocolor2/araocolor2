# 로그인 프로젝트 계획서 2 (Clerk 없이)

## 기술 스택

| 영역 | 선택 | 역할 |
|---|---|---|
| 프레임워크 | Next.js 15 | 전체 웹 앱 |
| 인증 | Supabase Auth | 구글 로그인, 세션 관리 |
| 전화 인증 | Twilio Verify | SMS 인증 코드 |
| 결제 | Stripe | 카드 결제, 이메일 영수증 |
| 데이터베이스 | Supabase | 댓글, 좋아요, 사용자 데이터 |
| 알림 | Supabase Realtime | 실시간 알림 |
| 배포 | Vercel | 호스팅 |

---

## Clerk 대비 차이점

| 기능 | Clerk (1안) | Supabase Auth (2안) |
|---|---|---|
| 구글 로그인 | 설정만 | 설정 + 코드 약간 |

| 전화 인증 | 내장 | Twilio 별도 연동 |

| 난이도 | 쉬움 | 보통 |

## 사용자 시나리오

### 1단계 — 비로그인 방문
- 커뮤니티 글 목록, 댓글 수, 좋아요 수는 보임
- 글 클릭 시 로그인 페이지로 이동
- 상품 구매 버튼 클릭 시 로그인 페이지로 이동

### 2단계 — 로그인
- 구글 계정 클릭 한 번으로 로그인 (Supabase Auth OAuth)
- 비밀번호 입력 없음

### 3단계 — 아이디 등록
- 로그인 후 댓글 작성 시도 시 아이디 등록 요청
- Supabase `profiles` 테이블에 저장
- 아이디 등록 완료 후 댓글 작성 가능

### 4단계 — 알림
- **결제 테스트와 함께 커뮤니티 테스트 범위에서 적용**

### 5단계 — 상품 구매
- 구매 버튼 클릭 → Twilio SMS 인증
- 인증 완료 → Stripe 결제창 (카드 번호, 주소 입력)
- **결제 완료 신호 확인 후 구매 확정 처리**
- **결제 완료 → 구매 내역 반영 + 이메일 영수증 발송**

### 5-1단계 — 결제 예외 처리
- **결제창 취소/이탈: 미결제로 종료**
- **결제 실패: 실패 메시지 + 재시도 버튼**
- **환불 완료: 구매 권한 회수**

### 5-2단계 — 커뮤니티 테스트
- **게시글/댓글/좋아요 기본 흐름 확인**
- **비로그인/로그인/구매회원 권한 차등 접근 확인**
- **내 글 댓글·좋아요 알림 수신 및 읽음 처리 확인**

### 6단계 — 재방문
- 개인정보 설정에서 이메일 주소, 아이디 확인 가능
- 비밀번호 없음

## 구현 순서

| 순서 | 작업 | 서비스 |
|---|---|---|
| 1 | Next.js 15 프로젝트 생성 | - |
| 2 | Supabase Auth 구글 로그인 연동 | Supabase |
| 3 | 로그인 UI 직접 제작 | Next.js |
| 4 | profiles 테이블 생성 (아이디 등록) | Supabase |
| 5 | 로그인 게이팅 (글/구매 버튼) | Next.js 미들웨어 |
| 6 | **Twilio SMS 인증 연동 (첫 구매 1회 기준)** | Twilio Verify |
| 7 | **Stripe 결제 연동 + 쿠폰 기본 규칙 적용** | Stripe Checkout |
| 8 | **결제 완료/실패/취소/환불 처리 규칙 반영** | Stripe + Supabase |
| 9 | **구매 내역/다운로드 반영 확인** | Supabase |
| 10 | **결제 테스트 케이스 실행 (정상/실패/취소/중복 클릭/쿠폰 오류)** | 통합 |
| 11 | **커뮤니티 테스트 케이스 실행 (글/댓글/좋아요/권한/알림)** | Supabase + Realtime |
| 12 | **결제-커뮤니티 권한 연동 테스트 (구매회원 접근/환불 회수)** | Stripe + Supabase |
| 13 | 개인정보 설정 페이지 | Supabase |

## 결제 테스트 체크 기준

| 항목 | 확인 내용 |
|---|---|
| 정상 결제 | **결제 완료 후 구매 내역이 바로 반영되는지 확인** |
| 결제 취소 | **결제창에서 취소 시 주문이 생성되지 않는지 확인** |
| 결제 실패 | **실패 메시지/재시도 버튼이 정상 동작하는지 확인** |
| 중복 클릭 | **결제 버튼 연속 클릭 시 중복 주문이 막히는지 확인** |
| 쿠폰 예외 | **만료/중복/조건 미충족 쿠폰 차단 확인** |
| SMS 인증 | **인증 실패/재시도 흐름 확인** |
| 환불 반영 | **환불 완료 시 구매 권한이 회수되는지 확인** |

## 커뮤니티 테스트 체크 기준

| 항목 | 확인 내용 |
|---|---|
| 게시글 작성 | **로그인 회원이 글 작성/수정/삭제 가능한지 확인** |
| 댓글/좋아요 | **댓글 작성, 좋아요 추가/취소가 즉시 반영되는지 확인** |
| 권한 구분 | **비로그인은 읽기 제한, 로그인 회원은 일반 게시판 작성 가능 여부 확인** |
| 구매회원 전용 접근 | **상품 구매 후 전용 게시판 접근 가능 여부 확인** |
| 환불 연동 | **환불 완료 후 전용 게시판 접근 차단 여부 확인** |
| 알림 동작 | **내 글 댓글/좋아요 발생 시 알림 뱃지 및 읽음 처리 확인** |
